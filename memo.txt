================================================================================
WxO移行計画（2026-03-20更新）
================================================================================

■ 現状のアーキテクチャ（VM上）
- Streamlit UI → APIサーバー → OpenAI Agents SDK → MCP経由でツール呼び出し
- MCPサーバー: visualize_data, visualize_prediction, forecast_linear, forecast_time_series, forecast_curve_fit
- MongoDB MCPサーバー: データ検索（find, aggregate）
- 全てDockerコンテナで動作

■ WxOのツール種別（4種類）
1. Pythonツール: @toolデコレータでWxO上で実行
2. MCPサーバー: リモートMCPサーバーをそのまま登録
3. OpenAPI: REST APIのOpenAPI仕様を登録
4. Langflow: ビジュアルフロー構築（今回不使用）

■ 確定事項
- 検索はリモートMCP必須（MongoDB MCPサーバーをそのまま活用）
- フロントは埋め込みチャットHTML（receiveイベントでHTML検出→iframeパネル表示）

================================================================================
アーキパターン（検索はリモートMCP共通、グラフ部分が異なる）
================================================================================

■ パターン2: 全部リモートMCP

  埋め込みチャット
    ↓ ↑ HTMLテキスト検出→iframeパネル表示
  WxO エージェント
    ↓ MCPプロトコル
  VM(nginx)
    ├── MongoDB MCPサーバー → MongoDB（検索）
    └── グラフMCPサーバー → Plotly生成 → HTMLテキスト返却

  実装コスト: 小（HTMLテキスト返却に変更するだけ）
  Prophet/scipy: VM上で動く（確実）
  リスク: HTMLテキストのメッセージサイズ上限

■ パターン3: ハイブリッド（検索MCP + グラフPythonツール）

  埋め込みチャット
    ↓ ↑ HTMLテキスト検出→iframeパネル表示
  WxO エージェント
    ├── リモートMCP → VM(nginx) → MongoDB MCPサーバー → MongoDB（検索）
    └── WxO Pythonツール → Plotlyグラフ生成 → HTMLテキスト返却

  実装コスト: 中（グラフ生成ツールをPythonツールとして書き直し）
  Prophet/scipy: WxO環境で動くか未検証
  リスク: ライブラリ制約、タイムアウト

■ パターン4: S3経由

  埋め込みチャット
    ↓ ↑ URL検出→iframeのsrcに設定
  WxO エージェント
    ↓
  Pythonツール or リモートMCP
    ├── データ取得（MongoDB）
    ├── Plotlyグラフ生成
    └── bytes返却 → WxOがS3に自動アップ → presigned URL

  実装コスト: 中
  メッセージサイズ: 問題なし（URLだけ）
  リスク: presigned URLの有効期限

================================================================================
ネットワーク構成
================================================================================
- VMにはnginxがあり、特定URLは外部アクセス可能
- ngrokは不要。nginxにリバースプロキシ設定を追加すればWxOから直接アクセス可能
  例: location /mcp { proxy_pass http://localhost:8011/mcp; }
- 認証はnginxレベルでBasic認証 or IPホワイトリスト

================================================================================
セキュリティ
================================================================================
- MongoDB直接公開は避ける
- REST API（FastAPI）を間に挟んでAPIキー認証をつけるのがベスト
- nginx経由ならnginxレベルでも認証可能

検証の優先順位（推奨）
================================================================================
1. [2-3] WxO Pythonツールから外部HTTP → ネットワーク疎通確認（全ての前提）
2. [2-2] リモートMCPでMongoDB MCPサーバーを登録 → 検索の実現
3. [3-1] Plotlyグラフ生成 → requirements.txtで入るか
4. [3-2/4-1] HTMLテキスト返却→iframe表示 → エンドツーエンド確認
5. [3-4/3-5] S3方式 → HTMLテキスト方式がサイズ上限に引っかかった場合の代替
